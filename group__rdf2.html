int rdf2Danalysis_AA(std::string path, std::vector<double>* rdfValues, molSys::PointCloud<molSys::Point<double>, double>* yCloud, double cutoff, double binwidth, int firstFrame, int finalFrame)
#include <src/include/internal/rdf2d.hpp>

Parameters
path in The file path of the output directory to which output files will be written.
rdfValues in Vector containing the RDF values.
yCloud in The input PointCloud.
cutoff in Cutoff for the RDF. This should not be greater than half the box length.
binwidth in Width of the bin for histogramming.
firstFrame in The first frame for RDF binning.
finalFrame in The final frame for RDF binning.

Calculates the in-plane RDF for quasi-two-dimensional water, when both the atoms are of the same type. The input PointCloud only has particles of type A in it. This is registered as a Lua function and is accessible to the user. Internally, this function calls the following functions:

  • rdf2::getSystemLengths (Gets the dimensions of the quasi-two-dimensional system).
  • ring::rdf2::sampleRDF_AA (Samples the current frame, binning the coordinates).
  • ring::rdf2::normalizeRDF (Normalizes the RDF).
  • ring::sout::printRDF (Writes out the RDF to the desired output directory, in the form of an ASCII file)

int normalizeRDF(int nopA, std::vector<double>* rdfValues, std::vector<int> histogram, double binwidth, int nbin, std::vector<double> volumeLengths, int nIter)
#include <src/include/internal/rdf2d.hpp>

Parameters
nopA in The number of particles of type A.
rdfValues in Radial distribution function values for all the frames, in the form of a vector.
histogram in The histogram for the current frame.
binwidth in The width of each bin for the RDF histogram.
nbin in The number of bins for the RDF.
volumeLengths in The confining dimensions of the quasi-two-dimensional system, which may be the slice dimensions or the dimensions of the box.
nIter in The number of iterations for which the coordinates will be binned. This is basically equivalent to the number of frames over which the RDF will be calculated.

Normalizes the histogram and adds it to the RDF. The normalization requires the plane area and height.
